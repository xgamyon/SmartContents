

position -----------------------------------------------------------------

* position 값은 부모 자식과의 연동보다
  먼저 선언된 상위의 가장 가까운 녀석을 포지션 값 부모로 생각한다.
  (fixed 는 윈도우 창 자체에서의 고정값이다 보니 제외된다.)

* relative 는 현재의 위치로부터 상하좌우로 움직이고

* absolute 는 그 하나의 영역 안에서 상하좌우로 움직인다.
  그래서 보통 홈페이지 제작시 relative 값이 먼저 부모로 적용 되어있고
  (absolute 가 적용되려면 부모인 position 값이 존재해야하는데 absolute는 영역이 있어야 움직이는데 처음에 존재하지 못하기에) 
  그 다음에 absolute 값이 존재한다!!




static :       디폴트값            
relative :
     상대적인,   현재 위치로부터!! (현재 위치에서 배치해주고 싶을때 쓴다.) 
                 (눈에 보이는 위치만 변경된것이지 실제 위치는 그대로 있다! )  
relative 에서의 
top        margin 값처럼 위에서 떨어져서 배치되는 형태로 보인다.
left        margin 값처럼 왼쪽에서 떨어져서 배치되는 형태로 보인다.
right      현재 위치에서 왼쪽으로 right 값만큼 움직인다.
bottom  현재 위치에서 위로 bottom 값만큼 움직인다.



absolute :
   절대적인,   붕 뜨긴 한데 float와 다르게 컨텐츠 영역도 무시하고 그냥 떠서 다닌다.  
                  width, height 값이 없을 때는 현재 위치에 붕 떠있다.
                  (margin의 auto값은 먹지 않고 상위에 가장 가까운 position 이 있을 경우 그녀석을 부모로 인식한다.)
absolute 에서의
top          브라우저 창을 기준으로 
left
right
bottom     브라우저를 기준으로 부모가 윈도우가 된다. 그래서 가장 하단에서 부터 값을 먹는다.


           
fixed :         고정적인  fixed 는 메뉴바,퀵메뉴 처럼 고정 형식처럼 쓰인다.
                  무조건 화면 단위에서 움직인다.
